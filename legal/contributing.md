# Contribuire a Dokyll

Grazie per il tuo interesse nel contribuire a Dokyll! 🎉

Ogni contributo è apprezzato, che si tratti di codice, documentazione, segnalazioni di bug, suggerimenti o miglioramenti del sito.

## Come contribuire

### Segnalare un bug

Prima di aprire una nuova issue:

1. Verifica che il problema non sia già stato segnalato.
2. Assicurati di utilizzare una versione recente di Dokyll.
3. Utilizza il template **Bug Report** o **Crash Report**.

### Richiedere una funzionalità

Hai un'idea per migliorare Dokyll?

Apri una issue utilizzando il template **Feature Request** e descrivi:

* Il problema che vuoi risolvere
* La soluzione proposta
* Eventuali alternative considerate

## Contribuire con codice

### 1. Fork del repository

Crea un fork del repository e clonalo:

```bash
git clone https://github.com/dokyll/dokyll.git
```

### 2. Crea un branch

Usa un branch dedicato:

```bash
git checkout -b feature/nome-feature
```

oppure

```bash
git checkout -b fix/nome-bug
```

### 3. Apporta le modifiche

Mantieni il codice:

* Pulito
* Leggibile
* Documentato
* Compatibile con le versioni supportate

### 4. Testa le modifiche

Verifica che:

* Dokyll si avvii correttamente
* Le build funzionino
* Non siano stati introdotti nuovi problemi

### 5. Commit

Scrivi messaggi di commit chiari.

Esempi:

```text
feat: add plugin loader
fix: resolve markdown parser crash
docs: improve installation guide
```

### 6. Apri una Pull Request

Descrivi chiaramente:

* Cosa cambia
* Perché cambia
* Eventuali effetti collaterali

## Linee guida

### Codice

* Segui lo stile del progetto.
* Evita dipendenze inutili.
* Mantieni le modifiche focalizzate.

### Documentazione

Le modifiche alla documentazione sono sempre benvenute.

Puoi contribuire:

* Correggendo errori
* Migliorando esempi
* Aggiungendo spiegazioni
* Traducendo contenuti

## Sicurezza

Non pubblicare vulnerabilità di sicurezza tramite issue pubbliche.

Se scopri una vulnerabilità di sicurezza, contatta i manutentori del progetto in privato fornendo tutti i dettagli necessari per riprodurla.

## Comportamento

Sii rispettoso con tutti i membri della community.

Non sono tollerati:

* Insulti
* Molestie
* Spam
* Comportamenti tossici

## Licenza

Contribuendo a Dokyll accetti che il tuo contributo venga distribuito sotto la stessa licenza del progetto.
